# Documentation Style

Casual and brief

## Voice

Write like you're explaining to a friend. No filler. Say the thing, stop.

## Structure

One idea per section. No intro that explains what you're about to explain. No summary that restates what you just said.

## Format

Code blocks for anything you'd type. No bold for emphasis. No emoji. No em dashes. No semicolons. Plain ASCII only. No →, use ->. No fancy quotes or special chars nobody types on a normal keyboard. Trim trailing whitespace. One trailing newline.
