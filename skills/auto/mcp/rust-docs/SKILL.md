# Rust Docs

Use the Rust MCP tools first for Rust questions.

- `crates_package_info` for crate versions and metadata (like latest version)
- `crates_search` for finding crates
- `crates_dependencies` for dependency trees
- `docs_rs_read` for docs.rs pages

Use this for:

- latest crate version
- crate metadata
- docs.rs pages
- feature flags
- dependency trees
- "what crate is this?"
- "what version is current?"
- after adding a crate / using it, ALWAYS check the docs for usage patterns

Never guess Rust crate versions.

ALWAYS lookup the docs on how to actually use the crate, never guess
Don't assume the api is like a similar crate, looks at the examples
