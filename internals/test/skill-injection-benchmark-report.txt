Date: 2026-05-05

What was tested
---------------
- Three local models from ./models:
  - Qwen3.6-35B-A3B-Claude-4.7-Opus-Reasoning-Distilled-APEX-I-Balanced.gguf
  - Qwen3.6-35B-A3B-Claude-4.7-Opus-Reasoning-Distilled-APEX-I-Quality.gguf
  - Qwen3.6-35B-A3B-uncensored-heretic.i1-Q5_K_M.gguf
- Injection modes:
  - no injection: HVA_RUNTIME_PROMPT_MODE=none, HVA_SKIP_ALL_INJECTS=1
  - soft injection: normal configured skill injection
  - hard injection: rust-style forced through HVA_HARD_INJECT_SKILLS
- Main benchmark asked each run to create a Rust project plus bingo-test-file.txt
  containing exactly 85.
- Llama was restarted for each benchmark run.
- Tokens/sec came from internals/healthcheck.sh --debug-cache.

The first hard-injection pass exposed a validator bug: the rust-style skill can
make a valid Cargo layout with src/lib.rs plus src/bin/*.rs instead of only
src/main.rs. The script now accepts both project structures. After that fix,
Balanced and Quality passed hard injection 5/5. Heretic still failed one hard
run behaviorally, so that is a model/injection result rather than just the old
test being too strict.

Winner / recommendation
-----------------------
Use the Quality model for this skill/tooling path. It was the only model that:
- passed no/soft/hard injection 5/5,
- passed the bigger project check,
- passed the stricter tool-proof run including the bingo sentinel,
- had the best eval throughput, around 90-94 tok/s in the matrix.

Balanced is stable on the simple matrix but slower and missed the bingo sentinel
in the stricter tool-proof prompt. Heretic is useful for comparison but less
reliable here: hard injection was 4/5, and it also missed bingo in the stricter
tool-proof prompt.

Bugs fixed while testing
------------------------
- check-system-prompt now verifies the actual project structure, not just that
  Pi returned text.
- check-system-prompt now verifies bingo-test-file.txt contains exactly 85.
- check-system-prompt now supports custom prompt templates, expected checks,
  fail-fast off for full benchmark matrices, kept dirs for inspection, and
  llama health/tokens-per-second output.
- llama restart/stop now waits for Docker --rm container removal before the
  next start, fixing a real name-race seen during model restarts.
- hva --stop now stops and removes HVA-managed helper containers.
- Generated benchmark project folders were removed after the runs.
- Version pins were updated and internals/check-versions.sh reports all ok.
